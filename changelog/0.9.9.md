# Version 0.9.9 release

Commit range: `f17ba3e..cca1169`

## Highlights
- Refactored build process to support inline scripts and conditional static file exports, replacing legacy HTML-embedded script/theme handling.
- Added RSS feed generation with absolute URL validation, content body handling, and publish configuration support.
- Added `footer_sort_by` configuration option with dedicated footer ordering logic.
- Introduced atomic file writing utilities and consolidated filesystem write paths for safer artifact output.
- Added multiple section format support in the `new` CLI workflow with improved section path normalization.
- Added raw HTML escaping with comprehensive link safety checks and configurable `allow_unsafe_html` opt-in.
- Added `theme-lock` configuration option with dynamic CSS adjustments for persistent theme selection.
- Refactored Typst HTML figure generation with improved structure and rendering consistency.
- Consolidated kaiti font family handling across CSS and font imports with multiple refinement passes.
- Restricted live-reload script injection to serve mode only.

## Behavior changes and migration notes
- Build process now exports static files conditionally based on `inline_script` configuration (defaults to `true`).
- Theme and mobile-TOC scripts are now inline-ready JS files instead of HTML-embedded blocks; `theme.html` replaced by `theme.js`.
- Live-reload script is only injected in serve mode, not in publish builds.
- Footer ordering now respects the `footer_sort_by` configuration field.
- Raw HTML in markdown content is now escaped by default; opt in via new `allow_unsafe_html` configuration.
- Link processing now performs safety checks (protocol validation, dangerous URI filtering).
- Theme selection can be locked via `theme_lock` configuration, disabling runtime theme toggling.
- Typst image handling renamed `html_figure` to `html_typst_figure` for clarity.
- New `Kodama.toml` sections support optional RSS publish configuration, `allow_unsafe_html`, and `theme_lock` fields.

## Features and improvements
- Added RSS feed generation (`compiler/rss.rs`) with feed metadata, item aggregation, and absolute URL validation.
- Added `footer_sort_by` configuration with ascending/descending footer ordering.
- Added multiple section format support in `cli/new.rs` with path normalization.
- Added atomic file writing for safer artifact and embed output.
- Added `relocate_trees_path` helper for centralized tree-path relocation.
- Added conditional live-reload script injection gated on serve mode.
- Added raw HTML escaping in markdown embedding with configurable filtering.
- Added link safety validation in content writing pipeline.
- Added `theme-lock` configuration with dynamic CSS class removal for locked theme mode.
- Improved Typst HTML figure generation structure and CSS consistency.
- Updated `kodama.typ` helper layout and `with-target-check` restructuring.
- Updated KaTeX version in math import template.

## Performance
- Consolidated script files and removed redundant HTML template blocks for theme/mobile-TOC.
- Simplified filesystem write paths through shared atomic write utilities.
- Removed dead `is_build` detection code from environment module.

## Stability and error handling

- Fixed CACHE_SCHEMA_VERSION regression (reverted to `schema-v1`).
- Fixed theme-options pseudo-elements rendering when empty.
- Improved raw HTML sanitization covering inline and block-level dangerous content.
- Strengthened link safety with protocol validation in content processing.
- Improved atomic write error handling across build artifacts and embedded markdown output.
